Investigación sobre GitHub

¿Qué es GitHub?

GitHub es una plataforma basada en la web que utiliza el sistema de control de versiones Git para alojar y gestionar proyectos de software. Permite a desarrolladores colaborar en proyectos, hacer seguimiento de cambios en el código, manejar diferentes versiones y coordinar el trabajo en equipo.

Ventajas de GitHub

  • Colaboración sencilla: múltiples usuarios pueden trabajar simultáneamente en un proyecto sin perder control de versiones.
  • Control de versiones: permite revertir cambios y manejar diferentes ramas (branches) para desarrollo paralelo.
  • Integración con herramientas: soporte para CI/CD, gestión de incidencias, wikis y más.
  • Hosting gratuito: para proyectos públicos y sitios web estáticos mediante GitHub Pages.
  • Amplia comunidad: millones de desarrolladores y miles de proyectos open source.

Desventajas de GitHub

  • Curva de aprendizaje: Git y GitHub pueden ser difíciles para principiantes.
  • Privacidad limitada en planes gratuitos: los repositorios privados tienen restricciones en planes gratuitos.
  • Dependencia de conexión a internet: para trabajar colaborativamente se requiere conexión constante.
  • Puede ser complejo para proyectos muy grandes: la gestión de muchos archivos o repositorios puede complicarse.

Datos Curiosos sobre GitHub

  • GitHub fue lanzado en 2008 por Tom Preston-Werner, Chris Wanstrath y PJ Hyett.
  • En 2018, Microsoft adquirió GitHub por 7.5 mil millones de dólares.
  • GitHub aloja más de 100 millones de repositorios activos (a 2024).
  • Git, el sistema de control de versiones en que se basa GitHub, fue creado por Linus Torvalds, el creador de Linux.
  • GitHub ofrece un sistema llamado "Actions" para automatizar tareas como pruebas y despliegues.

¿Cómo subir una página a GitHub para hostearla?

Aquí te explicamos paso a paso cómo subir una página web estática para publicarla usando GitHub Pages:

  1. Crea un repositorio nuevo en tu cuenta de GitHub. Puedes nombrarlo como quieras.
  2. Clona el repositorio en tu computadora, o usa la opción de subir archivos desde la interfaz web.
  3. Agrega tu página web, generalmente al menos un archivo index.html.
  4. Haz un commit y sube los cambios al repositorio.
  5. Activa GitHub Pages en la configuración del repositorio:
    • Ve a la pestaña Settings (Configuración) del repositorio.
    • Busca la sección Pages.
    • Selecciona la rama main (o master) y la carpeta raíz (root) para publicar.
    • Guarda los cambios y espera unos minutos.
  6. Accede a tu página web desde la URL que GitHub te proporcionará, usualmente https://tu-usuario.github.io/nombre-del-repositorio/.

Ejemplo usando Git en la terminal:

git clone https://github.com/tu-usuario/nombre-del-repositorio.git
cd nombre-del-repositorio
# Coloca aquí tus archivos web (index.html, css, js, etc)
git add .
git commit -m "Agrega página web"
git push origin main

Luego configuras GitHub Pages en la configuración del repositorio y listo, tu sitio estará en línea.
